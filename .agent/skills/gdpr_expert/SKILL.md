# GDPR Expert Skill

## Key Principles

| Principle | Meaning |
|-----------|---------|
| Lawfulness | Legal basis for processing |
| Purpose Limitation | Collect for specified purposes |
| Data Minimization | Only collect what's needed |
| Accuracy | Keep data correct |
| Storage Limitation | Don't keep longer than needed |
| Integrity | Secure processing |

## Lawful Bases for Processing

1. **Consent** - Freely given, specific, informed
2. **Contract** - Necessary for contract performance
3. **Legal Obligation** - Required by law
4. **Vital Interests** - Protect life
5. **Public Task** - Official authority
6. **Legitimate Interests** - Business need (with balance test)

## User Rights

| Right | Response Time |
|-------|---------------|
| Access | 30 days |
| Rectification | 30 days |
| Erasure ("Right to be Forgotten") | 30 days |
| Data Portability | 30 days |
| Object to Processing | Without delay |
| Withdraw Consent | Without delay |

## Consent Requirements

✅ Must be:
- Freely given (no penalty for refusing)
- Specific (per purpose)
- Informed (clear explanation)
- Unambiguous (affirmative action)
- Withdrawable (easy to revoke)

❌ Cannot be:
- Pre-ticked boxes
- Bundled with T&Cs
- Implied from silence

## Privacy Policy Must Include

- Identity of data controller
- Contact of DPO (if applicable)
- Purposes and legal basis
- Categories of data
- Recipients/transfers
- Retention periods
- User rights
- Right to complain to authority

## Data Breach Response

1. **Within 72 hours**: Notify supervisory authority
2. **Without undue delay**: Notify affected individuals (if high risk)
3. **Document**: All breaches in internal register

## Penalties

- Up to **€20 million** or **4% of global revenue**
- Whichever is higher

## When to Apply
Use when handling EU user data, creating privacy policies, or ensuring GDPR compliance.
